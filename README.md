## 무엇을 해결하려 했나

집에 뭔가 고장 났을 때, 관리사무소나 수리업체에 전화로 증상을 설명하는 일은 생각보다 어렵다. 특히 어르신이나 전문 용어를 모르는 사람에게는 "보일러가 어떻게 이상한지"를 말로 옮기는 것 자체가 장벽이다.

수리수리는 이 접수 과정을 **사용자가 편한 방식을 고르게** 만든 실험이다:

- 📷 **사진으로 진단받기** — 고장 부위를 촬영하면 접수
- 🎤 **음성으로 진단받기** — 말로 설명하면 접수
- 📋 **직접 문제 선택하기** — 12가지 항목에서 바로 선택

접수 후에는 방문 주소·연락처·희망 시간을 입력하면 전문가 배정 흐름으로 이어진다. 긴급 상황용 SOS 버튼도 상단에 상시 노출.

## 프로젝트 성격 (정직하게)

이 앱은 **화면 흐름과 UX를 검증하기 위한 프로토타입**이다. 카메라·음성 인식·기사 매칭은 실제 API 연동이 아닌 시뮬레이션으로 구현되어 있다. 서버·데이터베이스 없이 `index.html` 파일 하나로 동작한다.

- 검증하려던 것: "접수 방식을 3갈래로 나누면 비숙련 사용자의 진입 장벽이 낮아지는가"
- 다음 단계로 간다면: Web Speech API(음성), 카메라 캡처, 실제 접수 백엔드 연결
